# Future Ventures — Financial Model

## Scope of This Document

Future Ventures itself has no independent revenue or budget line — it is an evaluation function, not an operating venture. This document covers how *evaluation* is resourced. Once an idea graduates, its own financial model lives in its new project or business-unit directory (see, for example, [`../recoverhub/financial-model.md`](../recoverhub/financial-model.md) as the template pattern).

## Cost of Running the Pipeline

Evaluation cost is primarily Chief Research Officer and CFO time — no dedicated budget line exists yet. As pipeline volume grows, this should be made an explicit line item in [Bhubesi Ventures' financial model](../../business-units/bhubesi-ventures/financial-model.md) under shared-services cost allocation.

## Financial Criteria Applied at Evaluation

Ideas moving from "logged" to "Ready" (see [`sops.md`](./sops.md) Section 2) should have at least a rough answer to:

- What would Phase 1 cost, order of magnitude?
- What's the plausible revenue model (see the pattern in graduated ventures like [RecoverHUB](../recoverhub/financial-model.md) or [360Sports](../360sports/financial-model.md))?
- Does this require capital Bhubesi International doesn't currently have, implying an external deal structure per [`../../business-units/bhubesi-ventures/strategy.md`](../../business-units/bhubesi-ventures/strategy.md)?

None of the 8 ideas currently in the pipeline (see [`README.md`](./README.md)) have this analysis done yet — it's the next step for each before any can move to "Ready."

## KPIs

Feeds into [`executive-brain/kpi-framework.md`](../../executive-brain/kpi-framework.md):

- Pipeline size and conversion rate (logged → Ready → graduated).
- Time from logging to a go/no-go decision.

## Review

Reviewed monthly at the executive portfolio review and quarterly at the formal strategic review, per [`README.md`](./README.md).
