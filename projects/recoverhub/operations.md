# RecoverHUB — Operations

## Org Roles Involved

| Role/Seat | Contribution |
|---|---|
| [COO](../../ai-agents/workforce/coo.md) | Cross-unit coordination, quarterly planning cadence |
| Bhubesi Ventures (see [`../../business-units/bhubesi-ventures/`](../../business-units/bhubesi-ventures/README.md)) | Business model, partnership structuring, funding |
| [Bhubesi Labs / CTO](../../ai-agents/workforce/cto.md) | Platform/technical build once scoped |
| [Bhubesi Creative / CCO](../../ai-agents/workforce/chief-creative-officer.md) | Brand and user-facing design |
| [CFO](../../ai-agents/workforce/cfo.md) | Budget, grant financial terms, partner contract pricing |
| [Chief Legal Officer](../../ai-agents/workforce/chief-legal-officer.md) | Data-sharing agreements, safeguarding compliance, partner contracts |
| Program facilitators (human, not yet hired) | Curriculum delivery per [`sops.md`](./sops.md) |

## Operating Rhythm

- **Weekly:** partnership pipeline and cohort status check-in (COO-led).
- **Monthly:** outcome data review against the measure agreed with each partner.
- **Quarterly:** full review against [`strategy.md`](./strategy.md) objectives, per [`executive-brain/quarterly-planning-framework.md`](../../executive-brain/quarterly-planning-framework.md).

## Systems and Tools

Not yet built. Anticipated needs, to be scoped as part of a future project brief:

- **CRM** (a planned [Bhubesi OS](../bhubesi-os/README.md) module) for partner and referral pipeline management.
- **Document Management** (planned Bhubesi OS module) for data-sharing agreements and consent forms.
- A participant data system meeting the safeguarding/privacy bar set in [`sops.md`](./sops.md) — this is a Type 1 technical decision for the [CTO seat](../../ai-agents/workforce/cto.md) given the sensitivity of the data involved, not a default off-the-shelf choice.

## Partner and Vendor Management

Each active partner has a named internal owner (initially Bhubesi Ventures) responsible for the relationship, escalation per [`sops.md`](./sops.md) Section 5, and reporting outcomes into the review rhythm above.

## Capacity Constraints

At current (early-development) stage, RecoverHUB has no dedicated hired staff — delivery capacity is the binding constraint on how many partners/cohorts can be onboarded per quarter. This should be made explicit in every partnership conversation rather than over-committing.
