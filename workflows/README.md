# Workflows

This directory defines the repeatable processes Bhubesi International uses to execute work consistently, whether performed by a human or an AI agent.

## Contents

- [`standard-workflow.md`](./standard-workflow.md) — the default seven-step process for any non-trivial task, taken directly from `CLAUDE.md`.
- [`decision-making.md`](./decision-making.md) — how decisions get made and escalated, in practice.
- [`project-kickoff.md`](./project-kickoff.md) — the process for taking an idea from proposal to an active, approved project.

## Principle

Workflows exist so that output is consistent regardless of who (or what) performs the work. Before improvising a process for a task, check whether a workflow already covers it. If none does and the task is likely to recur, create one here rather than solving it ad hoc each time.
