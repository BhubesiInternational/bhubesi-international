# Documentation

This directory documents the operating system itself — how Bhubesi International's repository, governance, and AI-assisted workflows are structured and used.

## Contents

- [`architecture.md`](./architecture.md) — How the repository is organized and why, and how the pieces relate to one another.
- [`governance.md`](./governance.md) — Decision rights, approval structure, and how the Executive Office oversees business units and projects.
- [`getting-started.md`](./getting-started.md) — Onboarding guide for new team members and collaborators.
- [`glossary.md`](./glossary.md) — Shared terminology used across the company.

## Audience

This documentation is written for:

1. **Company leadership and staff**, who need to understand how work is organized and where to find or place information.
2. **AI agents (Claude sessions)**, which use this documentation alongside `CLAUDE.md` to determine role, scope, and process before acting.

Keep documentation here current. If a process or structure changes, update the relevant file in the same change — this repository is only useful if it reflects reality.
