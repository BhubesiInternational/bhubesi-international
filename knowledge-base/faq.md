# FAQ

## Where do I find the company's overall structure?

See [`../docs/architecture.md`](../docs/architecture.md) for the full model, or [`company-overview.md`](./company-overview.md) for a quick summary.

## Who decides what?

See [`../docs/governance.md`](../docs/governance.md) for the decision-rights matrix and escalation process.

## How do I propose a new project or venture?

Use [`../templates/project-brief-template.md`](../templates/project-brief-template.md) and follow [`../workflows/project-kickoff.md`](../workflows/project-kickoff.md).

## Which AI role should handle my task?

See [`../ai-agents/README.md`](../ai-agents/README.md) for the role-selection table.

## Is there application code in this repository?

Not yet. This repository currently defines organizational structure and documentation. Product/technical implementation for approved projects will be added under `projects/` or `business-units/bhubesi-labs/` once scoped and approved — see each project's README for status.

## Where do brand assets live?

Formal brand guidelines don't exist yet — see [`brand-guidelines.md`](./brand-guidelines.md) for interim guidance and ownership.

## How do I record a decision or outcome?

Use [`../templates/decision-log-template.md`](../templates/decision-log-template.md); see [`../workflows/decision-making.md`](../workflows/decision-making.md) for when this is required.

## Something doesn't fit the existing structure — what do I do?

Don't create a new top-level convention unilaterally. Raise it as a governance question per [`../docs/governance.md`](../docs/governance.md).
